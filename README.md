* **ภารกิจที่ 2: สร้างและรัน Spatial Model**
  * กำหนดพื้นที่ศึกษา (ภาคเหนือ) โดยใช้ ee.FeatureCollection
  * ดึงข้อมูลปัจจัยต่าง ๆ Fire Hotspot (MODIS FIRMS), NDVI (Sentinel-2), Elevation + Slope (SRTM), Wind (ERA5)
  * Normalize factors
  * Inverse factors บางตัว
  * ให้ Weight ทำ AHP
  * ทำ Consistency Ratio (CR)
  * เอาปัจจัยเข้าโมเดล
  * Normalize risk
  * คำนวณ percentileที่ 25,50,75 เพื่อใช้ในการ Classification
  * สร้าง Risk Map
 ผลลัพธ์ที่ได้ คือ ค่าความเสี่ยงการสะสม PM2.5 ระดับต่ำ-ปานกลางกระจายทั่วภาคเหนือ มีพื้นที่เสี่ยงสูง-สูงมาก ได้แก่ ชายแดนไทย-เมียนมา จ.แม่ฮ่องสอน, ตอนใต้และตอนบนของจ.เชียงใหม่, ตอนบนของจ.ลำพูน, ตอนล่างของจ.แพร่ และตอนกลางของจ.น่าน


* **ภารกิจที่ 3: Sensitivity Analysis**
  
  * ปรับ weight ±20%
  * ดูความแตกต่าง (Uncertainty) พบ Uncertainty อยู่ในระดับ robust และ moderate กระจายทั่วภาพ
  * ดูพื้นที่ Sensitive พบพื้นที่ Sensitive บริเวณรอยต่อทางตะวันตกเฉียงใต้ระหว่างจ.แม่ฮ่องสอนกับจ.เชียงใหม่
  * เปรียบเทียบโมเดลทั้ง 3 ตัว ผลคือ โมเดล sensitive ต่อการลด fire มากกว่าการเพิ่ม

* **ภารกิจที่ 4: Validation กับข้อมูลจริง**
  * จากค่า Correlation ≈ 0.5717 แสดงให้เห็นความสัมพันธ์ระดับปานกลางค่อนไปทางดี พื้นที่ที่โมเดลให้ risk สูง มีแนวโน้ม PM2.5 สูงจริง กล่าวคือโมเดลสามารถอธิบาย spatial pattern ของ PM2.5 ได้ค่อนข้างดีแต่ยังมีบางพื้นที่ที่ Sensitive

2. fire เป็นตัวขับหลัก

**คำถามสำหรับใช้คิดและตอบใน README**

**1. ถ้ามีข้อมูล Ground Truth จริง คิดว่า Accuracy ของโมเดลนี้จะอยู่ในระดับใด? เพราะอะไร?**

ถ้ามีข้อมูล Ground Truth จริง คิดว่า Accuracy ของโมเดลนี้จะอยู่ในระดับปานกลางเหมือนเดิม เพราะถึงโมเดลนี้จะจับปัจจัยด้านสิ่งแวดล้อมที่สำคัญของ PM2.5 ได้ เช่น กิจกรรมไฟป่า ลม การปกคลุมของพืช และลักษณะภูมิประเทศ แต่ไม่ได้รวมแหล่งกำเนิดมลพิษอื่น ๆ  เช่น การจราจรและมลพิษจากอุตสาหกรรม


**2. ปัจจัยที่ไม่ได้ใส่ในโมเดลแต่น่าจะสำคัญมีอะไรบ้าง? ทำไมถึงไม่ใส่?**

มีหลายปัจจัย เช่น ปริมาณการปล่อยมลพิษจากการจราจร กิจกรรมทางอุตสาหกรรม ความหนาแน่นของประชากร สภาพของชั้นบรรยากาศระดับล่าง รวมไปถึงมลพิษข้ามพรมแดนจากประเทศเพื่อนบ้าน เหตุผลที่ไม่ได้ใส่เพราะมีข้อจำกัดที่ขาดข้อมูลเชิงพื้นที่ที่มีความละเอียดสูง และอาจนำมาใช้ไม่ได้จริงในบริบทของภาคเหนือ

**3. โมเดลนี้ใช้ได้ดีในระดับ Scale ใด (ตำบล / อำเภอ / จังหวัด) เพราะอะไร?**

เหมาะสำหรับการวิเคราะห์ในระดับภูมิภาค เพราะ Resolution ของ Input data เช่น ข้อมูลลม ERA5, FIRMS มีความละเอียดค่อนข้างต่ำ ไม่เหมาะกับ scale ระดับจังหวัด

**4. ถ้าต้องนำโมเดลนี้ไปอัปเดตปีละครั้ง ขั้นตอนใดที่ต้องทำซ้ำและขั้นตอนใดที่คงที่?**

ส่วนใหญ่ต้องทำซ้ำเกือบทั้งหมดเพราะ PM 2.5 เป็นปรากฏการณ์ที่มีความ dynamic ปัจจัยต่าง ๆ ต้องอัปเดตอย่างสม่ำเสมอเพื่อให้สะท้อนภาพปัจจุบัน เช่น การรวบรวมข้อมูล การประมวลผลเบื้องต้นของ Input factors รวมไปถึงการ Normalized แต่ก็มีขั้นตอนที่คงที่ได้ เช่น โครงสร้างของโมเดล ปัจจัยที่เลือก และวิธีการถ่วงน้ำหนัก (ที่ได้มาจาก AHP) ยกเว้นว่าจะมีหลักฐานทางวิทยาศาสตร์ใหม่ที่ชี้ให้เห็นถึงการเปลี่ยนแปลงความสำคัญของปัจจัย
